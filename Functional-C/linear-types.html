Linear C

The first essay introduced functional connectives from C, {⊗ & → * ** ⊕ ∅}. The second essay showed how classical logic semantics with statements {⊤ ⊥ ;} make imperative programming in C more expressive. This essay develops a bookkeeping method with functions as object/type implications, inspired by linear logic. [Linear Logic, Di Cosmo et al]

For ledger keeping purposes, there will be dual forms of boolean values based on dual integer types, whole numbers 𝕎 and whole numbers ℕ. A value in 𝕎 provides a measure of falseness while a value in ℕ provides a measure of material truth.

𝕎 ℕ
0000 | ∅ ⊤
1111 | ⊥ 1

Variables have an assiciated count of type 𝕎 or ℕ depending on whether it is first assigned or instatiated. The count can be operated upon singly by linear negation (^⊥), which swaps the 𝕎 or ℕ type of count and by exponentials { ?predec !preinv postinc! postinv? }.
