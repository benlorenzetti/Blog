include <pthread.h> compile commands for GNU C++ compiler: g++ -pthread all pthread routines begin with pthread_ and are seperated into four type: thread management, mutexes, condition variables, and synchronization. Opaque objects pervades the design of the API An ANSI/POSIX standard for C. Therefore, it is portable. Starting Threads (Thread Management): pthread_create( thread, attr, start_routine, arg ); @param: an opaque, unique identifier for the new thread returned by the subroutine @param: attr, an opaque attribute object can be used or use NULL for defaults @param: a C routine that the thread will execute once it is created @param: a single argument can be passed to the function. It must be passed by reference as a pointer cast to type void. if you want to change the attributes, you have to work with attr objects: pthread_attr_init( attr ); pthread_attr_destroy( attr ); Stopping Threads (Thread Management) There are four primary ways a thread can be terminated: 1) the thread returns normally from its start_routine 2) the thread makes a call to: pthread_exit( status ); - note: if you call pthread_exit() from main, it will block main() until all the created threads have returned. 3) the thread is canceled by another thread: pthread_cancel( thread ); 4) the entire process is terminated (if main() exits or returns). If you are on Linux, this function can be used to determine which core a thread is running on: #include <sched.h> int sched_getcpu( void ); returns core number or -1 on error.
