How to use the Apache Server with C++ to Dynamically Write HTML Content

How to use the Apache CGI (Common Gateway Interface)

Apache allows you to write external programs to generate HTML content. This allows you to create webpages which can change based on nearly anything; user input, time, region, etc. Apache defines a Common Gateway Interface (CGI) to facilitate these external programs. The CGI is surprisingly easy to use and it is very powerfull. You can write the external program in nearly any language you are familiar with.

Configure Apache to Load the CGI Module on Start Up

First, configure the HTTP server to load the CGI module. The module has already been compiled during installation but it needs to configured to load on start-up.

  • [ec2-user@ip...]$ sudo /usr/local/apache2/bin/apachectl -k stop
  • [ec2-user@ip...]$ cd /usr/local/apache2/conf
  • [ec2-user@ip...]$ sudo vim httpd.conf

The configuration file should now be open for editing. Scroll down until you find the commented-out line "#LoadModule cgid_module modules/mod_cgid.so". Press 'i' to enter VIM's insert mode. Remove the comment hash so that the cgi module will be loaded. The line should now read "LoadModule cgid_module modules/mod_cgid.so". Press 'esc' to exit VIM's insert mode. Type ":w" and hit enter to save the changes.

Configure Apache to Permit CGI

Next, tell Apache that all files residing in the cgi-bin directory are CGI scripts and should be run accordingly. You should still have the httpd.conf file open in VIM.

Scroll down until you find the ScriptAlias directive: ScriptAlias /cgi-bin/ "/usr/local/apache2/cgi-bin/". Press 'i' to enter VIM's insert mode. Remove the quotation marks around the cgi-bin file path. After this the line should read: ScriptAlias /cgi-bin/ /usr/local/apache2/cgi-bin/. Press 'esc' to exit VIM's insert mode. Type ":w" and hit enter to save the changes. Type ":quit" and hit enter to exit VIM.
